# 싱크탱크 의사결정 지원 플랫폼 — 목업 설계

날짜: 2026-07-23
단계: 클릭 가능한 목업 (목 데이터 / 목 분석, 백엔드 없음)

## 1. 제품 비전 (전체 — 지금 다 만들지 않음)

거시 이슈에 대해 유저가 분기 시나리오를 놓고 토론·채택하고, 채택된 시나리오의 영향·결과를 다관점으로 예측해 의사결정을 돕는 플랫폼.

4개 독립 서브시스템:
1. **이슈 큐레이션 엔진** — 뉴스·SNS·싱크탱크 보고서·공공발표 취합 → AI가 거시 이슈 선정 (데이터 파이프라인)
2. **시나리오 토론 플랫폼** — 유저가 분기 시나리오 제출 → GitHub 이슈식 토론·투표 → 채택 시나리오 + 정리된 근거 저장 (인증 + DB + 커뮤니티)
3. **디지털 트윈 지도** — 채택 시나리오를 지도에서 선택 (지리 시각화)
4. **AI 영향·결과 분석** — 시나리오 + 근거 → 정치·경제·군사·민간 4관점 예측 (LLM)

## 2. 이번 단계 범위

**클릭 가능한 목업.** 전체 흐름을 한 번 관통하는 정적 프로토타입. 실제 수집·LLM·인증·영속 저장 **없음**. 모든 데이터·분석은 목.

목표: 흐름·화면·개념을 눈으로 검증. 무거운 파이프라인/트윈/LLM은 이후 단계.

## 3. 스택

- 순수 HTML / CSS / 바닐라 JS. 빌드 없음, 프레임워크 없음.
- 목 데이터는 `data.js` 하나에 JSON으로.
- 화면당 정적 HTML 파일을 늘리지 않음 — 상세/토론/분석은 **단일 템플릿 + 쿼리 파라미터**(`scenario.html?id=`)로 `data.js`를 렌더.
- 지도는 플레이스홀더: 스타일된 SVG 지도 또는 정적 지도 이미지 + 핫스팟.
- 그래프 시각화 라이브러리 없음 — 분기 트리는 CSS 들여쓰기 + 연결선으로 표현.

## 4. 화면 (6 + 공용 셸)

**공용 셸:** 상단 고정 nav `[대시보드] [지도]` (둘 다 대등한 최상위 진입점) + 로그인 유저 배지(목).

1. **이슈 대시보드** (홈, `index.html`) — AI 선정 거시 이슈 카드. 각 카드: 제목·요약·출처·트래픽·heat 배지. 클릭 → 이슈 상세.
2. **이슈 상세** (`issue.html?id=`) — 이슈 설명 + 제출된 분기 시나리오 **들여쓰기 트리**(부모-자식) + "시나리오 제출" 버튼. 시나리오 0개 빈 상태 포함.
3. **시나리오 제출 폼** (`submit.html?issue=`) — 제목·설명·분기 근거 입력 + 부모 시나리오 선택(선택 사항). 제출 → (목) 토론 화면으로.
4. **시나리오 토론** (`scenario.html?id=`) — 시나리오 본문 · "X에서 분기됨" breadcrumb · 평면 코멘트 스레드 + 찬반 투표 · **"채택으로 표시" 버튼** · 정리된 근거 박스 · 여기서 분기된 하위 시나리오 목록.
5. **디지털 트윈 지도** (`map.html`) — 지도 + 채택 시나리오 핫스팟 + 선택 패널 → "AI 분석 실행" 버튼. 채택 시나리오 0개 빈 상태 포함.
6. **AI 분석 뷰** (`analysis.html?id=`) — 정치·경제·군사·민간 4관점 결과 패널 (목 분석) + "분석 중…" 로딩 연출.

## 5. 데이터 모델 (목, `data.js`)

- **issue**: `id, title, summary, sources[], traffic, heat`
- **scenario**: `id, issueId, parentScenarioId(nullable), title, body, rationale, votes{up,down}, adopted(bool), mapLocation`
- **comment**: `id, scenarioId, author, body, ts`
- **analysis**: `scenarioId, lenses{political, economic, military, civilian}` (각 텍스트)

이슈당 **여러 시나리오 채택 허용** — "여러 미래를 동시에 놓고 비교"가 브랜칭의 취지. 지도 핫스팟 = 채택된 시나리오.

## 6. 흐름

대시보드 → 이슈 상세 → (제출 폼 | 시나리오 토론) → 채택 → 지도에 핫스팟 등장 → 지도에서 선택 → AI 분석 4관점.

두 개의 최상위 진입점(대시보드 / 지도)은 nav로 항상 접근.

## 7. Out-of-scope (이번 단계 제외)

- 실제 인증/로그인 플로우 (로그인 유저 배지만 목)
- 대댓글(답글의 답글) — 1단계 평면 코멘트 + 투표로 충분
- 실제 GIS 지도, 실시간 협업/웹소켓
- 그래프 트리 시각화 라이브러리 (CSS 들여쓰기로 충분)
- 실제 뉴스 수집·LLM·영속 저장

## 8. 검증 (목업이라도)

- 6개 화면 모두 링크로 도달 가능하고 흐름이 끊기지 않음.
- 빈 상태(시나리오 0, 채택 0) 렌더 확인.
- 분기 부모-자식 관계가 트리로 보임.
